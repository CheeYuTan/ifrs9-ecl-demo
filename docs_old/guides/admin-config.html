Admin Configuration

Configure data sources, column mappings, LGD parameters, and application settings.

Setup Wizard

The setup wizard runs on first launch and handles initial configuration. It can be re-run at any time:

EndpointDescription
GET /api/setup/statusCheck if setup is complete
POST /api/setup/validate-tablesValidate that required Unity Catalog tables exist
POST /api/setup/seed-sample-dataCreate tables and seed sample data
POST /api/setup/completeMark setup as complete
POST /api/setup/resetReset setup status to re-run the wizard

Configuration Sections

The admin configuration is organized into sections. Each section can be read and updated independently:

Data Sources

Configure Unity Catalog table paths and Lakebase schema settings:

GET /api/admin/config/data_sources

{
  "catalog": "main",
  "schema": "ifrs9",
  "lakebase_schema": "expected_credit_loss",
  "lakebase_prefix": "lb_",
  "tables": {
    "model_ready_loans": "main.ifrs9.model_ready_loans",
    "loan_ecl_weighted": "main.ifrs9.loan_ecl_weighted",
    "loan_level_ecl": "main.ifrs9.loan_level_ecl",
    "portfolio_ecl_summary": "main.ifrs9.portfolio_ecl_summary",
    "ifrs7_stage_migration": "main.ifrs9.ifrs7_stage_migration"
  }
}

LGD Parameters

Configure Loss Given Default by product type:

PUT /api/admin/config/lgd_parameters
{
  "Mortgages": {"lgd": 0.15, "cure_rate": 0.30},
  "Personal Loans": {"lgd": 0.45, "cure_rate": 0.15},
  "Auto Finance": {"lgd": 0.35, "cure_rate": 0.20},
  "Credit Cards": {"lgd": 0.80, "cure_rate": 0.10},
  "SME Lending": {"lgd": 0.55, "cure_rate": 0.12}
}

Scenario Configuration

Configure macroeconomic scenarios and their default weights:

PUT /api/admin/config/scenarios
{
  "base": {"weight": 0.50, "pd_multiplier": 1.0, "lgd_multiplier": 1.0},
  "upside": {"weight": 0.20, "pd_multiplier": 0.8, "lgd_multiplier": 0.9},
  "downside": {"weight": 0.30, "pd_multiplier": 1.5, "lgd_multiplier": 1.2}
}

Column Mapping

If your source tables use different column names, configure mappings:

Auto-Suggest Mappings

GET /api/admin/suggest-mappings/model_ready_loans

Uses fuzzy matching to suggest column mappings based on name similarity.

Validate Mappings

POST /api/admin/validate-mapping
{
  "table_key": "model_ready_loans",
  "mappings": {
    "loan_id": "account_number",
    "product_type": "product_category"
  }
}

Validate with Type Checking

POST /api/admin/validate-mapping-typed
{
  "table_key": "model_ready_loans",
  "mappings": {
    "loan_id": "account_number",
    "gross_carrying_amount": "balance"
  }
}

Validates both column existence and data type compatibility.

Data Discovery

EndpointDescription
GET /api/admin/available-tablesList all tables in the configured schema
GET /api/admin/table-columns/{table}Get column names and types for a table
GET /api/admin/table-preview/{table}Preview first N rows of a table
GET /api/admin/schemasList available schemas
GET /api/admin/discover-productsDiscover product types from the data
GET /api/admin/auto-detect-workspaceAuto-detect Databricks workspace settings
POST /api/admin/auto-setup-lgdAuto-configure LGD parameters from data

Connection Testing

POST /api/admin/test-connection

Tests the Lakebase connection and returns connection details (host, database, user, SSL status).

Seeding Defaults

POST /api/admin/seed-defaults

Seeds default configuration values including chart of accounts, RBAC users, and LGD parameters. Safe to run multiple times (uses upsert logic).

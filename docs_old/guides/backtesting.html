Backtesting

Validate model performance using the traffic-light framework with industry-standard metrics.

Backtesting dashboard — discrimination metrics with traffic-light status

Traffic-Light Framework

Each backtest metric is evaluated against thresholds and assigned a traffic-light color:

MetricGreenAmberRedDirection
AUC (Area Under ROC Curve)≥ 0.700.60 – 0.70< 0.60Higher is better
Gini Coefficient≥ 0.400.20 – 0.40< 0.20Higher is better
KS (Kolmogorov-Smirnov)≥ 0.300.15 – 0.30< 0.15Higher is better
PSI (Population Stability Index)≤ 0.100.10 – 0.25> 0.25Lower is better
Brier Score≤ 0.150.15 – 0.25> 0.25Lower is better

The overall traffic light for a backtest is determined by the worst individual metric result.

Running a Backtest

POST /api/backtest/run
{
  "model_type": "PD",
  "config": {
    "observation_window": "2024-Q1 to 2024-Q2",
    "outcome_window": "2024-Q3 to 2024-Q4"
  }
}

The backtest engine:

  1. Pulls loan-level data from the portfolio
  2. Computes predicted vs. actual default rates by cohort
  3. Calculates all five metrics (AUC, Gini, KS, PSI, Brier)
  4. Assigns traffic-light colors based on thresholds
  5. Stores results in the backtest results table

Viewing Results

List All Backtests

GET /api/backtest/results?model_type=PD

Get Specific Backtest

GET /api/backtest/{backtest_id}

Returns the full backtest result including:

  • Overall traffic light (Green/Amber/Red)
  • Individual metric values and traffic lights
  • Cohort-level predicted vs. actual rates
  • Configuration and metadata

Trend Analysis

Monitor how model performance changes over time:

GET /api/backtest/trend/PD

Returns a time series of backtest results for the specified model type, enabling trend analysis and early detection of model degradation.

Interpreting Results

Overall ResultAction Required
GreenModel performing well. No action needed. Continue monitoring.
AmberModel showing signs of degradation. Investigate root cause. Consider recalibration.
RedModel not fit for purpose. Immediate recalibration or replacement required. Escalate to model governance.
Regulatory Expectation Regulators expect backtesting to be performed at least quarterly. Results should be documented and any Amber/Red outcomes should have documented remediation plans.

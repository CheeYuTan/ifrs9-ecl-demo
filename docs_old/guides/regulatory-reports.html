Regulatory Reports

Generate IFRS 7 disclosures and regulatory reporting packages.

Generating a Report

POST /api/reports/generate/{project_id}
{
  "report_type": "ifrs7_disclosure",
  "user": "regulatory.reporting@bank.com"
}

The response includes the full report data structured for display and export.

IFRS 7 Disclosure Package

The IFRS 7 disclosure report includes the following sections (mapped to IFRS 7 paragraphs):

Credit Risk Exposure (IFRS 7.35M)

Maximum exposure to credit risk by product type and stage, showing GCA and ECL coverage ratios.

Stage Migration Matrix (IFRS 7.35I)

A 3×3 matrix showing how loans moved between stages during the reporting period, with loan counts and GCA amounts.

Loss Allowance Reconciliation (IFRS 7.35H)

Opening-to-closing reconciliation of the loss allowance (ECL provision) by stage, showing:

  • Opening balance
  • New originations
  • Derecognitions
  • Stage transfers (net impact)
  • Model parameter changes
  • Macroeconomic scenario changes
  • Management overlays
  • Write-offs
  • Closing balance

Sensitivity Analysis (IFRS 7.35G)

Impact on ECL of changes to key assumptions (PD and LGD shifts of ±10%, ±20%).

Report Lifecycle

Draft ──▶ Finalized

Listing Reports

GET /api/reports?project_id=Q4-2025-ECL&report_type=ifrs7_disclosure

Finalizing a Report

POST /api/reports/{report_id}/finalize

Finalized reports cannot be modified. They serve as the official regulatory submission record.

Exporting to CSV

GET /api/reports/{report_id}/export

Returns the report data as a downloadable CSV file for import into external systems.

Attribution Waterfall The ECL movement report uses the attribution engine to decompose ECL changes into their constituent drivers. This is generated via GET /api/data/attribution/{project_id} and provides the data for the waterfall chart in the UI.

Model Registry

Version control, governance, and lifecycle management for PD, LGD, and EAD models.

Model Lifecycle

Every model in the registry follows a governed lifecycle:

Draft ──▶ Validated ──▶ Approved ──▶ Champion │ │ │ ▼ └──────────────────────────────────▶ Retired
StatusDescriptionWho Can Transition
DraftNewly registered model, not yet validatedAny analyst
ValidatedModel has been backtested and meets performance thresholdsModel validator
ApprovedModel approved for production use by model governance committeeManager / approver
ChampionCurrently active production model (one per model type + product)Manager (via promote)
RetiredNo longer in use, preserved for auditManager

Registering a Model

POST /api/models
{
  "model_name": "PD Mortgage Model v3",
  "model_type": "PD",
  "algorithm": "XGBoost",
  "version": 3,
  "description": "Mortgage PD model using XGBoost with macro variables",
  "product_type": "Mortgages",
  "cohort": "2024-Q4",
  "parameters": {
    "n_estimators": 200,
    "max_depth": 6,
    "learning_rate": 0.05
  },
  "performance_metrics": {
    "auc": 0.82,
    "gini": 0.64,
    "ks": 0.38,
    "psi": 0.05
  },
  "training_data_info": {
    "n_samples": 25000,
    "n_features": 12,
    "training_period": "2020-01 to 2024-06"
  },
  "created_by": "model.developer@bank.com"
}

Updating Model Status

PUT /api/models/{model_id}/status
{
  "status": "validated",
  "user": "model.validator@bank.com",
  "comment": "Backtesting passed with Green traffic light"
}

Promoting to Champion

When a model is approved, it can be promoted to champion. This automatically retires the previous champion for the same model type and product.

POST /api/models/{model_id}/promote
{
  "user": "head.of.risk@bank.com"
}

Comparing Models

Compare performance metrics across multiple models side-by-side:

POST /api/models/compare
{
  "model_ids": ["mdl-001", "mdl-002", "mdl-003"]
}

Returns all models with their parameters, performance metrics, and status for comparison.

Audit Trail

Every status change is recorded in the audit trail:

GET /api/models/{model_id}/audit

Returns a chronological list of all actions (registration, validation, approval, promotion, retirement) with who performed each action and when.

Model Types

TypeDescriptionKey Metrics
PDProbability of Default modelsAUC, Gini, KS, PSI
LGDLoss Given Default modelsR², RMSE, MAE
EADExposure at Default modelsR², RMSE, CCF accuracy
StagingStage assignment modelsAccuracy, migration stability
SatelliteMacroeconomic satellite modelsR², forecast accuracy
Governance Requirement Models must be in "approved" or "champion" status to be used in production ECL calculations. Draft and validated models can be used for testing and comparison only.

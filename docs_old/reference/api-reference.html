API Reference

Complete REST API documentation for all 80+ endpoints.

Base URL All endpoints are relative to the application root: https://<your-app-url>. All request/response bodies use JSON (Content-Type: application/json).

Health

GET /api/health

Check application and Lakebase connectivity.

Response

{"status": "healthy", "lakebase": "connected", "rows": 1}

Projects & Workflow

GET /api/projects

List all ECL projects.

Response

Array of project objects with project_id, project_name, current_step, created_at, etc.

GET /api/projects/{project_id}

Get a specific project by ID.

Path Parameters

project_idstringProject identifier

Response

Full project object including current_step, audit_log, overlays, scenario_weights, signed_off_by.

POST /api/projects

Create a new ECL project.

Request Body

project_idstring, requiredUnique project identifier
project_namestring, requiredDisplay name
project_typestring, default "ifrs9"Project type
descriptionstringProject description
reporting_datestringReporting period end date (YYYY-MM-DD)
POST /api/projects/{project_id}/advance

Advance the project to the next workflow step.

Request Body

actionstring, requiredStep action name
userstring, requiredUser performing the action
detailstringAdditional detail/comment
statusstring, default "completed"Step completion status
POST /api/projects/{project_id}/overlays

Save management overlays for a project.

Request Body

overlaysarray, requiredArray of overlay objects: {id, product, type, amount, reason, ifrs9}
commentstringComment (also advances the workflow if provided)
POST /api/projects/{project_id}/scenario-weights

Save scenario probability weights. Weights must sum to 1.0.

Request Body

{"weights": {"base": 0.50, "upside": 0.20, "downside": 0.30}}
POST /api/projects/{project_id}/sign-off

Sign off the ECL calculation.

Request Body

{"name": "Jane Smith, CRO"}
POST /api/projects/{project_id}/reset

Reset project back to Step 0. Preserves audit log.

Simulation

POST /api/simulate

Run Monte Carlo ECL simulation.

Request Body

n_simulationsinteger, default 1000Number of iterations (100–5000)
pd_lgd_correlationfloat, default 0.30PD-LGD correlation
aging_factorfloat, default 0.08Stage migration aging factor
pd_floorfloat, default 0.001Minimum PD
pd_capfloat, default 0.95Maximum PD
lgd_floorfloat, default 0.01Minimum LGD
lgd_capfloat, default 0.95Maximum LGD
scenario_weightsobject, optionalCustom scenario weights

Response

Returns ecl_by_product, scenario_summary, loss_allowance_by_stage, ecl_by_scenario_product, and run_metadata.

GET /api/simulation-defaults

Get default simulation parameters, available scenarios, and product types.

POST /api/simulate-stream

Run simulation with Server-Sent Events (SSE) for real-time progress updates. Same request body as /api/simulate.

SSE Events

progressSimulation progress updates
resultFinal simulation results
errorError message if simulation fails
keepaliveHeartbeat to keep connection alive
POST /api/simulate-validate

Validate simulation parameters before running. Returns errors, warnings, estimated runtime, and memory usage.

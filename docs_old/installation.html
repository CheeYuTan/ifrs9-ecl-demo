Installation

Deploy IFRS 9 ECL Workspace to your Databricks environment.

One-Command Setup For the fastest path to running locally: 
git clone <repo-url> && cd "Expected Credit Losses"
cp .env.example .env    # Edit with your Databricks credentials
./install.sh            # Installs deps, builds frontend, runs tests
cd app && python app.py # Start the server on :8000

Prerequisites

Before you begin, ensure you have the following:

  • A Databricks workspace on AWS, Azure, or GCP with Unity Catalog enabled
  • A Lakebase database instance provisioned in your workspace
  • Unity Catalog catalog and schema with loan portfolio data (or use sample data)
  • Databricks CLI installed and authenticated (databricks auth login)
  • Python 3.10 or later
  • Node.js 18+ (only if building the frontend from source)
  • Workspace admin or CAN_MANAGE permission on Databricks Apps
Lakebase Provisioning If you don't have a Lakebase instance yet, create one from the Databricks workspace UI under SQL → Lakebase, or via the CLI: 
databricks lakebase create-database-instance \
  --name ifrs9-ecl-demo-db \
  --capacity SMALL

Step-by-Step Installation

1

Clone the Repository

Clone the IFRS 9 ECL Workspace repository to your local machine:

git clone https://github.com/your-org/ifrs9-ecl-workspace.git
cd ifrs9-ecl-workspace
2

Configure Lakebase Connection

The application connects to Lakebase automatically when deployed as a Databricks App (via auto-injected PGHOST, PGDATABASE, PGUSER, PGPASSWORD environment variables). For local development, set the Lakebase instance name:

export LAKEBASE_INSTANCE_NAME="ifrs9-ecl-demo-db"
export DATABRICKS_CONFIG_PROFILE="your-profile"

The backend will use the Databricks SDK to generate OAuth tokens and connect automatically.

3

Build the Frontend (Optional)

A pre-built frontend is included in app/static/. To rebuild from source:

cd app/frontend
npm install
npm run build
cd ../..

The build output is placed in app/static/ and served by FastAPI.

4

Deploy as a Databricks App

Deploy the application using the Databricks CLI:

databricks apps deploy ifrs9-ecl \
  --source-code-path ./app

Configure the app resource to attach your Lakebase instance. In the Databricks UI, go to Compute → Apps → ifrs9-ecl → Resources and add a Lakebase resource pointing to your database instance.

5

Run the Setup Wizard

On first launch, the application displays a setup wizard that:

  1. Tests the Lakebase connection
  2. Creates all required database tables and schemas
  3. Validates that Unity Catalog source tables are accessible
  4. Seeds default configuration (chart of accounts, RBAC users, etc.)
  5. Optionally loads sample data for demonstration

You can also trigger setup via the API:

curl -X POST https://<app-url>/api/setup/seed-sample-data
curl -X POST https://<app-url>/api/setup/complete \
  -H "Content-Type: application/json" \
  -d '{"user": "admin"}'
6

Configure Data Sources

Navigate to the Admin panel in the application to configure:

  • Unity Catalog tables — map your catalog.schema.table paths
  • Column mappings — map your column names to the expected schema
  • LGD parameters — configure Loss Given Default by product type
  • Scenario weights — set base/upside/downside scenario probabilities

Verification

After deployment, verify the application is running correctly:

Health Check

curl https://<app-url>/api/health

Expected response:

{
  "status": "healthy",
  "lakebase": "connected",
  "rows": 1
}

Setup Status

curl https://<app-url>/api/setup/status

Should return "setup_complete": true after the wizard finishes.

List Projects

curl https://<app-url>/api/projects

Should return an array (empty if no projects created yet).

Local Development

To run the application locally for development:

# Terminal 1: Backend
cd app
pip install -r requirements.txt
python app.py

# Terminal 2: Frontend (hot-reload)
cd app/frontend
npm install
npm run dev
OAuth Token Refresh The backend includes a background thread that refreshes the Lakebase connection pool every 45 minutes. OAuth tokens expire after ~60 minutes, so this ensures uninterrupted operation. If you see authentication errors, the pool will automatically reinitialize.

Troubleshooting

IssueSolution
PGHOST not setEnsure the Lakebase resource is attached to the Databricks App, or set LAKEBASE_INSTANCE_NAME for local dev.
password authentication failedOAuth token expired. The auto-refresh thread should handle this. Restart the app if persistent.
relation does not existRun the setup wizard or call POST /api/setup/seed-sample-data to create tables.
Frontend shows blank pageEnsure app/static/ contains the built React app. Run npm run build in app/frontend/.
Jobs fail to triggerEnsure Databricks Jobs are provisioned via POST /api/jobs/provision and notebook paths are correct.

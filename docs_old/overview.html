What is IFRS 9 ECL Workspace and why does it exist?

What is IFRS 9 ECL Workspace?

IFRS 9 ECL Workspace is an end-to-end Expected Credit Loss platform built natively on the Databricks Lakehouse. It replaces legacy SAS-based ECL systems with a modern, cloud-native architecture that combines:

  • A React single-page application for the user interface — an 8-step guided workflow that takes credit risk teams from data ingestion through to sign-off.
  • A FastAPI backend that orchestrates all business logic, model execution, and reporting.
  • Lakebase (managed PostgreSQL) for transactional state — workflow progress, model registry, GL journals, approval workflows, and configuration.
  • Unity Catalog for governed loan portfolio data, satellite model outputs, and ECL results.
  • Databricks Jobs for heavy compute — satellite model training, ECL calculation, and data synchronization.

Supported Data Sources

The application reads loan portfolio data from Unity Catalog tables. The following input tables are required:

TableDescriptionSource
model_ready_loansLoan-level data with PD, LGD, EAD, staging, and borrower attributesUnity Catalog
loan_ecl_weightedScenario-weighted ECL at loan levelECL calculation pipeline
loan_level_eclPer-scenario ECL at loan levelECL calculation pipeline
portfolio_ecl_summaryAggregated portfolio-level ECL by product, stage, and scenarioECL calculation pipeline
ifrs7_stage_migrationPeriod-over-period stage transition countsData processing pipeline
Flexible Schema Mapping The Admin Configuration panel allows you to map your existing column names to the expected schema — no data transformation required. See the Admin Configuration guide.
